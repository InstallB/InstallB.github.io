算法笔记_倍增LCA

发表于 2019-05-26 | 更新于 2019-06-15

0.前言

我不会tarjan
我也不会RMQ
这篇文章写的是倍增求LCA(两个点的最近公共祖先)的方法
虽然不是最快的 但是足够了 而且也不难写

1.概念

给定一棵有根树
我们需要求两个点的最近(Lowest)公共(Common)祖先(Ancestor)
何为LCA?

  • 给定一棵有根树
  • 给定两个节点 a 和 b
  • 找到深度最大 且同时是 a 和 b 的祖先的点
    祖先 简单地说 就是从一个点 求它的父亲节点 再求它父亲节点的父亲节点 一路遍历到根节点 所有遍历到的节点都是这个点的祖先

算了 放张图吧

画太丑了
红线上的点代表 11 的祖先 蓝线上的点代表 12 的祖先
1 和 2 即为 11 和 12 的公共祖先
其中2是 11 和 12 的最近公共祖先

2.应用

有时候就是作为模板用的
LCA有一个特性 如果同一条边不允许重复经过 树上$a$到$b$的唯一路径是$a->LCA->b$
这不是废话吗
此外树上差分问题也需要LCA
对不起 我只知道这些
总之 想到要用 就去使用就可以了

3.实现

这里先说最朴素的算法
先通过dfs预处理每一个点的深度和父亲节点编号
如果两个点在同一深度 我们可以每次把这两个点分别找它们的父亲

  • 如果两个点不同 则继续找父亲的父亲 以此类推
  • 如果这时候两个点是同一个点 这个点一定是LCA
    实现的时候可以直接存两个变量 分别代表两个待求点的编号 每次把两个变量改成 自己代表编号 的父亲 的编号一步一步往上改

dp数组预处理
在dfs后面
父亲的父亲是祖父
祖父的祖父是曾曾祖父
以此类推
…… 

void init(){
    for(int i = 1;i <= 20;i ++){
        // 20 这里只要2的这个数次方超过树高就行了
        for(int j = 1;j <= n;j ++){
            dp[i][j] = dp[i - 1][dp[i - 1][j]];
        }
    }
}

2.查询

我们在把深节点向上遍历到和浅节点一个深度时
实际上是把深度差转化成二进制数
x位为1就把当前点改为这个点的$2^x$级父亲 为0不改
为了实现方便从高位往低位遍历 找了会找过头就不找 不然就改
比如深度相差9的时候
01001 16>9不 8<=9跳 4>1不跳 2>1不跳 1<=1不跳

第一步走完要先判断两点是不是相等
可能LCA是两个点其中的一个

第二步就是一起向上改
这里有点不一样
因为当两个点的$2^x$级父亲一样时 我们不能判断是LCA还是LCA的祖宗
于是找深度相同且最小的X,Y (X为x的祖先 Y为y的祖先 X,Y深度相同) 使得X!=Y
所以找完祖先一样不变 找完不一样就两个都向上改
最后找到的节点X/Y的父亲是LCA

int lca(int x,int y){
    if(dep[x] < dep[y]) swap(x,y);
    if(dep[x] != dep[y]){
        for(int i = 20;i >= 0;i --){
            if(dep[dp[i][x]] < dep[y]) continue;
            x = dp[i][x];
        }
    }
    if(x == y) return x;
    for(int i = 20;i >= 0;i --){
        if(dp[i][x] == dp[i][y]) continue;
        x = dp[i][x];
        y = dp[i][y];
    }
    if(x == y) return x;
    return dp[0][x];
}
